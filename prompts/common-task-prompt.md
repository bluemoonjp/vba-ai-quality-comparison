# 共通課題Prompt

以下は、全AI条件に同一内容で渡す共通promptです。仕様確認は1回まで、生成後の修正依頼は1回までです。

---

あなたはExcel VBAに詳しい開発者です。以下の仕様に基づき、月次クレーム集計ブックに埋め込むVBAマクロを作成してください。

## 実験上の制約

- 仕様確認1回、修正依頼1回の条件で実験します。
- 仕様確認は1回まで可能です。
- この依頼への回答では、VBAコードと説明を出してください。
- 生成後、こちらから修正依頼を出せるのは1回だけです。
- 実データではなく、公開可能な合成サンプルを前提にしてください。
- VBAコードは標準モジュールに貼り付けて使える形にしてください。
- VBAの識別子は英語中心を推奨します。説明は日本語で書いてください。

## 課題

月次ブック `月次クレーム集計YYMM.xlsm` に格納するVBAマクロを作成してください。

このマクロは、月次ブックと同じフォルダ配下の `daily\YYMM\` フォルダを再帰探索し、日次ブック `クレーム集計YYMMDD.xlsx` を読み込みます。

読み込んだ日次ブックのクレーム件数を、月次ブックの `月次集計` シートへ、支店コード + 業務コード + 日付単位で転記してください。

## 対象月の判定

- 月次ブック名は `月次クレーム集計YYMM.xlsm` 形式です。
- `YYMM` は月次ブック名から判定してください。
- 日次ファイル探索起点は `ThisWorkbook.Path\daily\YYMM\` です。
- 日次ファイル名は `クレーム集計YYMMDD.xlsx` 形式です。
- 日付は日次ファイル名から判定してください。
- 月次ブック名や日次ファイル名が想定形式でない場合は、処理対象外またはエラーとして分かるようにしてください。

## 日次ブック仕様

各日次ブックには `日次集計` シートがあります。

列構成:

| 列 | 見出し |
| --- | --- |
| A | `処理区分` |
| B | `支店コード` |
| C | `支店名` |
| D | `業務コード` |
| E | `業務名` |
| F | `クレーム件数` |
| G | `備考` |

`処理区分 = "ok"` の行だけを通常転記対象としてください。その他の行は異常系サンプルとして扱い、転記せずログ対象にしてください。

## 月次ブック仕様

月次ブックには以下のシートがあります。

- `月次集計`: 転記先
- `マスタ`: 支店/業務/小計グループの参照情報
- `異常系`: 異常系の確認またはログ用途
- `期待結果`: サンプル `.xlsx` に含まれる期待値シート。本番 `.xlsm` にはなくてもよいです。

`月次集計` の列構成:

| 列 | 内容 |
| --- | --- |
| A | 行種別 |
| B | 小計グループ |
| C | 地域 |
| D | 支店コード |
| E | 支店名 |
| F | 業務コード |
| G | 業務名 |
| H:AL | 1日から31日 |
| AM | 月合計 |
| AN | 備考 |

行種別:

- `DETAIL`: 転記対象行
- `SUBTOTAL`: 小計行。入力キーとして扱わないでください。

転記キーは `支店コード + 業務コード` です。支店名・業務名は表示または確認用です。

## 転記ルール

- 再実行しても二重計上されないようにしてください。
- 実行開始時に、`DETAIL` 行の H:AL の日別転記セルをクリアしてから再集計してください。
- 行ラベル、マスタ、備考、無関係シートは壊さないでください。
- 有効な日次行の件数を、該当する `DETAIL` 行と日付列へ加算してください。
- 同じ日・同じ支店コード・同じ業務コードが複数行ある場合は合算してください。
- AM列の月合計を再計算してください。
- 小計行は、同じ小計グループの `DETAIL` 行から再計算してください。
- 28日/30日の月では、月外の日付列は空欄のままにし、月合計にも含めないでください。

## 小計ルール

小計ルールはあえて少し変則的です。

- 大きい支店は、支店単独で小計を出します。
- 小さい支店は、地域単位で小計を出します。
- どちらも `小計グループ` 列で判定できます。
- `SUBTOTAL` 行は転記対象ではありません。
- 小計行は日別列と月合計列を持ち、同じ小計グループの `DETAIL` 行を集計します。

## エラー・ログ対象

以下は転記せず、ログまたは処理結果として分かるようにしてください。

- 未知の支店コード
- 未知の業務コード
- 月次表に存在しない支店コード + 業務コード
- 空欄のクレーム件数
- 数値ではないクレーム件数
- 同じ日付の重複日次ファイル
- ファイル名形式が違う日次ファイル
- 対象月と異なる日次ファイル
- `日次集計` シートがない日次ファイル

ログの出し方は設計して構いませんが、どこに何を記録するか説明してください。

## 参照設定

追加のVBA参照設定を使うかどうかは判断して構いません。

ただし、追加参照を使う場合は以下を説明してください。

- 参照設定名
- 使う理由
- Excel/VBEで必要な設定
- late binding または標準VBAだけで書く代替案

追加参照なしで実装しても構いません。

## 出力してほしいもの

1. 標準モジュールに貼り付ける完全なVBAコード
2. マクロの入口プロシージャ名
3. コードの配置場所
4. 実行方法
5. 参照設定の要否
6. エラー処理・ログ出力の説明
7. 主な前提と制限
8. `2602`, `2604`, `2605` のサンプルで確認すべきテスト観点
