# VBA課題仕様書

この文書は、AIへ渡す課題仕様の正本です。非プログラマーが業務上の困りごととして説明できる粒度を保ちます。

## やりたいこと

日ごとに作られるクレーム集計ファイルを、月次のクレーム集計表に転記するExcel VBAを作ってください。

日次ファイルには、支店コード、支店名、業務コード、業務名、クレーム件数が入っています。月次集計表には、支店+業務ごとの行と、1日から31日までの日別列、月合計列があります。

## ファイル

- マクロを入れる想定の月次ファイル: `月次クレーム集計YYMM.xlsm`
- サンプル月次ファイル: `samples/workbooks/月次クレーム集計2605.xlsx`
- 日次ファイル: `samples/workbooks/daily/2605/クレーム集計260501.xlsx` など
- 対象月: `2605`

## 日次ファイル

各日次ファイルには `日次集計` シートがあります。

| 列 | 見出し |
| --- | --- |
| A | 支店コード |
| B | 支店名 |
| C | 業務コード |
| D | 業務名 |
| E | クレーム件数 |

ファイル名の `YYMMDD` から転記する日付列を判断してください。

## 月次ファイル

月次ファイルには `月次集計` シートがあります。

| 列 | 見出し |
| --- | --- |
| A | 支店コード |
| B | 支店名 |
| C | 業務コード |
| D | 業務名 |
| E:AI | 1日から31日 |
| AJ | 月合計 |

支店コード + 業務コードが一致する行へ、日次ファイルの件数を転記してください。

## 必須ルール

- 転記前に、月次集計表の日別列と月合計をクリアしてください。
- 同じ日付、同じ支店+業務が複数行ある場合は合算してください。
- 転記後に月合計を計算してください。
- 同じファイルで2回実行しても二重加算しないようにしてください。
- MsgBoxで処理を止めないでください。結果やエラーはログシートか一覧形式で残してください。
- 外部通信、実パス固定、実データ前提の処理は禁止です。

## AIに求める出力

- 標準モジュールに貼り付けられる完全なVBAコード。
- マクロの入口プロシージャ名。
- コードの配置場所。
- 実行方法。
- 参照設定の要否。
- エラー処理とログ出力の説明。
- 再実行安全性の説明。

## 対象外

- 実際の `.xlsm` ファイルの作成。
- サンプルworkbookの変更。
- 実務データへの適用。
- 複雑な小計や異常系の網羅。
