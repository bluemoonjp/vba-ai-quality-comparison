## 評価ルール

| 観点 | 判定内容 |
| --- | --- |
| import | `final-vba.bas` を標準モジュールとして import できるか |
| run | 入口プロシージャを Excel 上で実行できるか |
| transfer | キー照合で対象日/対象列が期待値と一致するか |
| task_specific_check | 課題固有の主要結果が期待値と一致するか。月次集計なら `monthly_total` など、実験開始前に名称を固定する |
| rerun_safe | 同じ workbook セッションで再実行しても二重加算や破壊が起きないか |
| no_msgbox | `MsgBox` / `InputBox` などで自動検証が停止しないか |
| major_defect | 実行不能、集計不一致、二重加算、停止UI、実パス/実データ前提などの重大欠陥がないか |

必要に応じて、保守性は定性的観察として別欄に記録します。数値採点する場合は、実験開始前に基準を固定します。

## Excel COM 実行手順

1. 条件ごとに新しい scratch ディレクトリを作る。
2. 全条件で同じ sample workbook と期待値から開始する。
3. workbook を scratch 内で検証用形式へ一時保存する。`.xlsm` は成果物にしない。
4. `final-vba.bas` を import する。日本語を含む `.bas` が VBE import で文字化けする場合は、scratch 内に CP932 変換 copy を作り、それを import する。
5. `final-output.md` または `.bas` 内で入口プロシージャが明示されていればそれを使う。
6. 未明示の場合は、単一の引数なし `Public Sub` のみ実行候補にする。候補が複数またはゼロなら `blocked` と記録する。
7. 実行後、期待値と照合する。行順ではなくキーで比較する。
8. 同じ workbook セッションで再実行し、二重加算や不要な変更がないことを確認する。
9. 実行ログには公開可能な環境情報だけを残す。

## 環境注意

- クラウド同期フォルダ上で `ThisWorkbook.Path` が URL 形式になる場合は、一時ドライブやローカル scratch など公開可能な形で回避し、ログには一般化した対処だけを書く。
- ログにローカル絶対パス、ユーザー名、端末名、スクリーンショットを残さない。
- 検証中に出た Excel ダイアログは、内容を要約して記録する。画像は保存しない。
