# samples

全条件で共有する合成サンプルと期待結果を置きます。

## 推奨構成

- `source/`: サンプル生成元の CSV、JSON、スクリプト。
- `expected/`: 評価に使う期待値。行順ではなく安定したキーで照合できる形にする。
- `workbooks/`: 公開前チェック済みの sample workbook。
- `workbook-audit.md`: workbook 公開前チェック結果。

## サンプル差し替え手順

1. 実験目的に合わせて、架空データだけで source data を作る。
2. 全条件で同じ workbook と期待値を使えるように固定する。
3. 期待値は、行番号ではなく業務上のキーで照合できるようにする。
4. workbook を保存する場合は、ドキュメントプロパティ、隠しシート、名前定義、外部リンク、コメント、接続情報を確認する。
5. 変更したサンプルと期待値の関係をこの README または `workbook-audit.md` に記録する。

## 期待値の書き方

- キー列を明示する。例: `branch_code + business_code`。
- 照合対象列を明示する。例: `day_01`, `day_02`, `day_15`, `month_total`。
- 変更されてはいけない列がある場合は、検証計画にも書く。
- 未知キー、空白、非数値、重複、対象外日付などを扱う場合は、サンプル上で意図を明確にする。

## 保存しないもの

- 実データ由来の workbook。
- 未確認の Office バイナリ。
- 検証中に生成した `.xlsm`。
- ローカル絶対パスやユーザー名を含むログ。
