# prompt constraints

全条件で共通に適用する prompt 制約です。

## 禁止

- 実データ、実ローカルパス、ユーザー名、端末名、secret、private情報を使わない。
- 外部通信をしない。
- モデル成果物として `.xlsm` や scratch workbook を保存しない。
- サンプル workbook の構造を、課題仕様にない形で変更する前提にしない。
- 他条件の成果物、レビュー、評価結果を参照しない。
- 検証が停止する `MsgBox` / `InputBox` に依存しない。必要な通知はログやシート上の記録で代替する。

## 推奨

- 標準VBAまたは late binding を優先する。
- 追加参照が必要な場合は、参照名、理由、代替案を説明する。
- 入口プロシージャ名を `final-output.md` と `.bas` のどちらかで明示する。
- 再実行安全性を前提に、既存値の扱いとクリア範囲を説明する。
- エラー処理とログ出力方針を書く。

## 成果物

- `first-output.md`
- `final-output.md`
- `final-vba.bas`
- `notes.md`
- `correction-request.md`。修正依頼がない場合も、なしと分かるように記録する。

## 記録

確認質問が0回だった場合も `prompt-records/` に0回として記録します。修正依頼が0回だった場合も、後続レポートで改善度比較不能と判断できるように記録します。
