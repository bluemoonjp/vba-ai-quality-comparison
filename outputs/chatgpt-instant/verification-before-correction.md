# chatgpt-instant 修正前検証

## 検証対象

- `first-output.md`
- 静的確認のみ。Excel実行はIssue #6で行う。

## 良い点

- `ThisWorkbook.Path\daily\YYMM\` 配下を再帰探索する設計になっている。
- 月次ブック名と日次ファイル名から `YYMM` / `YYMMDD` を判定している。
- `支店コード + 業務コード` で `DETAIL` 行を引く設計になっている。
- 再実行前に既存転記値をクリアする発想がある。
- 小計行を `小計グループ` 単位で再計算しようとしている。
- 追加参照設定を必須にせず、late bindingでDictionaryとFileSystemObjectを使っている。

## 主な懸念

1. 重複日次ファイルの扱いが仕様と違う。
   - 仕様では、同一日付に複数ファイルがある場合、1ファイルだけ処理して重複をログする。
   - 初回出力では、複数ファイルがある日付を丸ごと転記しない。
   - 2026-04-02のように正常ファイルと重複ファイルがあるケースで、正常側の期待値も失われる。

2. 異常分類が粗い。
   - `処理区分 <> ok` をすべて `NON_OK_ROW` として記録している。
   - サンプルには `unknown_branch`, `unknown_business`, `blank_count`, `non_numeric_count`, `duplicate_file` などの異常区分があり、期待動作と比較するには分類が不足する。

3. ログに絶対パスを書き込む設計になっている。
   - `file.Path` をそのまま `異常系` シートへ出すため、実行環境のローカルパスがworkbook内に残る。
   - Public実験の安全線として、相対パスまたはファイル名/サブフォルダだけを記録する方がよい。

4. クリア対象が広い。
   - 仕様では、転記前に `DETAIL` 行の `H:AL` をクリアする。
   - 初回出力は `SUBTOTAL` 行と `AM` 列もクリアする。
   - 後段で再計算しているが、数式保持方針や月外列の空欄維持との関係が不安定。

5. `マスタ` シートの読み取りがサンプルレイアウトに合っていない。
   - `FindHeaderColumn` は1行目だけを探すが、サンプルの `マスタ` では支店マスタ見出しが2行目、業務マスタ見出しが12行目、支店・業務対応表が24行目にある。
   - 月次 `DETAIL` からのfallbackはあるが、仕様上のマスタ参照としては弱い。

## 修正依頼方針

1回だけの修正依頼では、仕様適合に直結する上記5点をまとめて指摘する。
