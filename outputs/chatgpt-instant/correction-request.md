# chatgpt-instant 修正依頼

以下の点を修正し、標準モジュールに貼り付けられるVBAコード全文を再提示してください。コード内容を省略せず、修正理由も簡潔に説明してください。

1. 同一日付の日次ファイルが複数ある場合、日付全体をスキップしないでください。安定した基準、たとえばフルパス昇順の先頭1件だけを処理し、残りを `DUPLICATE_DAILY_FILE` としてログしてください。
2. `処理区分 <> "ok"` の行も、可能な限り仕様上の異常分類に合わせてログしてください。未知支店、未知業務、空欄件数、非数値件数、重複ファイルなどが一律 `NON_OK_ROW` にならないようにし、元の `処理区分` もログで確認できるようにしてください。
3. `異常系` シートのファイル列に、実行環境の絶対パスを残さないでください。`ThisWorkbook.Path` からの相対パス、または `daily\YYMM\...` 形式の相対パスを記録してください。
4. 再実行時のクリアは、仕様どおり `DETAIL` 行の `H:AL` の日別転記セルを中心にしてください。`SUBTOTAL` 行と `AM` 列は最後に再計算または復元し、28日/30日の月外日列は空欄のまま、月合計対象外にしてください。
5. `マスタ` シートを読む場合は、サンプルレイアウトに合わせてください。支店マスタは2行目、業務マスタは12行目、支店・業務対応表は24行目に見出しがあります。少なくとも月次 `DETAIL` 行と矛盾しない形で、支店コード、業務コード、支店+業務の有効組み合わせを判定してください。

追加参照設定を使う場合は、参照設定名、理由、設定手順、late bindingまたは標準VBAだけの代替案も説明してください。
