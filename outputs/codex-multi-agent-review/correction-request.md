# codex-multi-agent-review 修正依頼

以下の点を修正してください。修正版では、標準モジュールに貼り付けられる完全なVBAコードを再掲してください。仕様の追加や前提変更はせず、既存仕様に合わせる修正だけを行ってください。

1. `マスタ` シートの読み取りをサンプルレイアウトに合わせてください。支店マスタは2行目、業務マスタは12行目、支店・業務対応表は24行目の見出しを前提にして、少なくとも支店コードと業務コードを正しく読めるようにしてください。見出しが見つからない場合は致命エラーにせず、ログに記録して月次 `DETAIL` 行からフォールバックしてください。
2. 再実行前のクリア範囲を修正してください。事前クリアは `DETAIL` 行の `H:AL` 日別転記セルだけに限定してください。`DETAIL` 行の月合計、`SUBTOTAL` 行の日別・月合計は、最後の再計算時に安全に更新してください。
3. `処理区分 <> "ok"` の日次行も、転記せずログに記録してください。支店コード、業務コード、日付、可能ならクレーム件数の元値も残してください。
4. 0件の月合計・小計・日別小計は、期待結果と比較しやすいように `0` を書くのではなく空欄にしてください。28日/30日の月外日列も空欄を維持してください。
5. 実行時ログに保存するファイルパスは、可能な限り `ThisWorkbook.Path` からの相対パスにしてください。`ThisWorkbook.FullName`、日次フォルダ、日次ファイルパス、重複ファイルの採用済みパスもフルパスのまま残さないようにしてください。
6. 月次ブックが未保存で `ThisWorkbook.Path` が空の場合は、分かりやすいメッセージで停止してください。
7. ログにクレーム件数の列を追加するか、同等に構造化された形で、空欄・非数値・未知キー・非ok行の元値が分かるようにしてください。
8. 修正版でも、追加参照設定を使わない場合は late binding を維持してください。追加参照が必要になる場合は、参照設定名、理由、設定方法、late binding の代替案を説明してください。
